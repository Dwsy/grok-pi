# Issue: [标题]

## Goal

让 `grok-pi` 的 `/resume` 与 Pi TUI 的会话选择语义一致：默认只加载当前目录会话；切到 All 后加载全局会话并按标题本地筛选；列表扫描不得阻塞 TUI。

## 背景/问题

当前 adapter 在打开 `/resume` 时同步解析整个 Pi 会话目录，并且始终发布全量 catalog。大量 JSONL 会占用 LocalSet，表现为 Resume 卡死；也没有 Pi TUI 的 Current Folder / All scope。

## 验收标准 (Acceptance Criteria)

- [ ] WHEN 打开 Pi `/resume`，系统 SHALL 默认显示当前工作目录的会话。
- [ ] WHEN 切换到 All tab，系统 SHALL 异步加载全局会话，且可按 Pi session title 筛选。
- [ ] IF 会话目录很大，THEN 扫描 SHALL 不阻塞 Pager 的事件循环。
- [ ] WHEN 异步 catalog 响应晚于 tab 切换，系统 SHALL 丢弃不属于当前 tab 的响应。

## 关键决策

| 决策 | 理由 |
|------|------|
| 复用 Pager 原生 Picker tabs | 保持 Grok Pager 为唯一终端 UI，不自绘 Pi selector。 |
| 默认 Current，All 按需加载 | 对齐 Pi TUI；避免首次全局解析会话文件。 |
| adapter 用 `spawn_blocking` 扫描 | JSONL 文件读取和解析不可占用 LocalSet，消除 UI 卡死。 |
