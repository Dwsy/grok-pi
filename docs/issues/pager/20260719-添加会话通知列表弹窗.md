# Issue: 添加会话通知列表弹窗

## Goal

为 Pi `ui.notify` 事件提供仅当前 `grok-pi` 进程有效、按活动会话隔离且支持文本/类型搜索的 Pager 原生弹窗。

## 背景/问题

现有 Pi notify 仅投影为 toast（info 另写入 scrollback），warning/error 无可回看列表；频繁或短暂的通知容易错过。

## 验收标准 (Acceptance Criteria)

- [x] WHEN Pi 发送 info、warning 或 error notify，系统 SHALL 保留该事件到对应活动会话的进程内列表。
- [x] WHERE 活动 Pi 会话中执行 `/notify`，系统 SHALL 打开 Pager 原生可搜索 modal，不污染 scrollback。
- [x] IF 会话切换或进程重启，THEN 通知不会泄漏到其他会话，且不持久化到 Pi session 文件。

## 实施阶段

### Phase 1: 规划和准备
- [x] 分析 Pi RPC notify → adapter → Pager 的真实调用链。
- [x] 确认用户选择：仅本次运行、收录全部 notify 类型。

### Phase 2: 执行
- [x] 追加按 session ID 分组的 Pager 进程内通知状态。
- [x] 新增 `/notify`、Action 与 dispatcher。
- [x] 复用 Pager picker/modal，实现搜索、键盘和鼠标交互。

### Phase 3: 验证

- [x] Adapter 测试：`cargo test -p pi-grok-adapter`（76）通过。
- [x] 代码审查：`git diff --check` 通过；仅审查本 Issue 文件。

## 关键决策

| 决策 | 理由 |
|------|------|
| 状态存于 `ExternalUiState` | Pager 是唯一可见 TUI；无需改 Pi session 格式或 adapter。 |
| 以 Pi session ID 分桶 | 同一进程中切换会话时不串通知。 |
| 重用原生 Picker modal | 保持 Grok Pager 唯一 TUI，天然支持搜索/键鼠。 |
