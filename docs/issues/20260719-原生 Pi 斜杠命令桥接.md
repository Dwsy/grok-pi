# Issue: 实验性 Pi 原生命令桥接

## Goal

以 `pi-*` 实验命令验证 Pi 原生 selector/lifecycle 能否在 `grok-pi` 的既有 Remote TUI 投影中运行；保持 Pager 是唯一可见终端表面、adapter 完全 headless。

## 背景/问题

当前裸 `/resume`、`/model` 已正确复用 Grok Pager 原生选择器和 Pi RPC 状态。用户希望同时实验 Pi interactive TUI 的原生 `SessionSelectorComponent` 与 `ModelSelectorComponent`，并评估 export/import/share/login/logout/reload 是否能同样由 Pi 本体执行。

## 验收标准 (Acceptance Criteria)

- [x] 梳理 Pi interactive 实现和 RPC 可达性。
- [x] 注入 `pi-model`、`pi-resume`、`pi-reload`，不修改 `pi-main`。
- [x] `pi-model`、`pi-resume` 仅通过已有 Remote TUI 投影渲染 Pi 原生组件。
- [x] RPC 不公开的 `pi-export`、`pi-import`、`pi-share`、`pi-login`、`pi-logout` 明确报告边界，不伪造实现。
- [x] 运行二进制 crate check、adapter tests 与目标 extension 测试，并审查 diff。

## 关键决策

| 决策 | 理由 |
|---|---|
| 裸 `/model`、`/resume` 保持 Pager 原生 | 既有实现已经通过 Pi RPC 正确驱动状态；不应倒退或双重实现。 |
| Pi selector 只暴露为 `pi-model`、`pi-resume` | 实验面与稳定命令隔离，命令行为可并行对照。 |
| 复用 `PI_GROK_REMOTE_TUI=1` | Pi RPC 的 `custom(factory)` 不可直接序列化；已有试验投影是唯一不改 Pi 源码的承载面。 |
| 其余命令先报告边界 | public RPC 只有 HTML export；没有 JSONL export/import、share 或 auth RPC，不能将 Grok 产品实现伪装为 Pi 原生。 |

## Pi 命令可达性矩阵

| Pi interactive 命令 | Pi RPC | `pi-*` 结果 |
|---|---|---|
| `model` | `get_available_models` + `set_model` | `pi-model`：Pi `ModelSelectorComponent` 经 Remote TUI。 |
| `resume` | `switch_session` | `pi-resume`：Pi `SessionSelectorComponent` 经 Remote TUI。 |
| `reload` | extension `ctx.reload()` | `pi-reload`：Pi 原生 reload lifecycle。 |
| `export` | 仅 `export_html` | `pi-export`：明确提示 JSONL export 未公开。 |
| `import` | 未公开 | `pi-import`：明确提示边界。 |
| `share` | 未公开 | `pi-share`：明确提示边界。 |
| `login` / `logout` | 未公开 | `pi-login` / `pi-logout`：明确提示边界。 |

## Notes

- Pi interactive 的 command dispatcher 位于 `pi-main/packages/coding-agent/src/modes/interactive/interactive-mode.ts`；原生 components 是同目录 `components/model-selector.ts` 与 `components/session-selector.ts`。
- `resume-x` 的关键模式是注册独立命令，调用 Pi component / `ctx.switchSession()`，不替换内建 `/resume`。
- `pi-grok-adapter` 不引入 TUI 依赖；extension 由 composition binary 写入 `NamedTempFile` 并由 Pi 进程加载。
