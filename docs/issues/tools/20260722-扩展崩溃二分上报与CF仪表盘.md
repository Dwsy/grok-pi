# Issue: 扩展崩溃二分上报与 CF 仪表盘

## Goal

用 **Bun+TS CLI** 对 Pi 扩展做 VSCode 风格二分排查，只上报 **扩展名 + 包目录名** 到
**Cloudflare Workers + D1**，并用静态仪表盘统计崩溃热榜与处理记录。

## 背景/问题

- `grok-pi` 已有启动期 `spawn_with_extension_self_heal` / `bisect_extension_culprit`
  （见 `crates/codegen/xai-grok-pager-bin/src/bin/grok-pi.rs`），但仅本地 `eprintln`，无聚合。
- 社区需要「哪些扩展在 grok-pi/Pi-RPC 下会炸」的匿名统计，**禁止**绝对路径、home、stack、会话。
- 栈选 CF 免费：Workers + D1 + Pages；客户端 Bun+TS CLI。

## 隐私边界（硬约束）

| 允许上报 | 禁止 |
|---|---|
| `ext_name`（basename / package name 短名） | 绝对路径、用户名、home |
| `package_dir`（如 `@scope/pkg` 或 `pi-pretty`） | 源码、stack、cwd、会话 |
| `kind`: `crash` \| `combo` \| `unknown` | 机器指纹作主键 |

服务端丢弃未声明字段；长度 cap；rate limit。

## 验收标准 (Acceptance Criteria)

- [x] WHEN CLI 对一组 `--extension` 路径做二分，系统 SHALL 输出仅含 `ext_name`/`package_dir`/`kind` 的 JSON
- [x] WHEN `report` 调用 Worker，系统 SHALL 只持久化白名单字段到 D1
- [x] WHERE 仪表盘打开，系统 SHALL 展示 Top 崩溃扩展、按日计数、triage 状态
- [x] IF 无法隔离单扩展，THEN kind=`combo` 且可不带 ext 或记为组合
- [x] IF 无 `REPORT_URL` / 上报失败，THEN CLI 不阻塞本地结果（probe 与 report 分离）
- [x] 不修改 Pi 源码；S1–S3 不强制改 grok-pi（S4 可选 env 默认 off）

## 关键决策

| 决策 | 理由 |
|------|------|
| 独立 `tools/ext-crash-telemetry`，不进 Rust crate | 与 Pager 解耦；Bun 迭代快；不触发 Cargo |
| 复刻二分语义，不调 grok-pi 内部 | 可单测；S4 再接 |
| 探针可注入（mock / 真实 `pi` spawn） | 默认 mock 测算法；真实探针可选 |
| D1 + 公开写 + 管理 secret | 免费；防刷用 token 可选 |
| 上报默认 opt-in | 隐私与网络可控 |
