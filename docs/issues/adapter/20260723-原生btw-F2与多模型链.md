# Issue: 原生 /btw F2 开关 + 多模型链

## Goal

在 **不改 Pi 源码、不桥 juicesharp/pi-btw 包** 的前提下，为 grok-pi 提供可选的原生 `/btw`：

1. F2 `pi_btw` **默认关**，开启后注入 Pi 扩展 + adapter `x.ai/btw`
2. 开启时 **自动 block** 冲突包：`npm:pi-btw`、`npm:@narumitw/pi-btw`、`npm:@juicesharp/rpiv-btw`
3. 可配置最多 **3 个模型** 链式 fallback；空 = 当前主会话模型
4. **同样改造 recap**：`recap_model` 升为最多 3 槽 fallback

## 边界

| 层 | 归属 |
|---|---|
| TUI | 原生 `BtwCommand` / `btw_overlay` / `BtwBlock`（已有） |
| Agent | Pi extension `complete()` 旁路，不写主会话 LLM 历史 |
| Adapter | `x.ai/btw` → bridge command；custom `pi-grok-btw/v1` → answer |
| Conflicts | `native_feature_conflicts.toml` + startup `enabled_native_features` |

## 方案

### F2 / 配置

| key | 默认 | 说明 |
|---|---|---|
| `pi_btw` | `false` | 总开关；restart_required；external_only |
| `btw_model` / `btw_model_2` / `btw_model_3` | `""` | 槽 1–3；空槽跳过；全空 → 当前会话模型 |
| `recap_model` / `recap_model_2` / `recap_model_3` | `""` | 同上，改造现有单槽 |

模型解析链（btw 与 recap 共用语义）：

```text
configured non-empty slots in order
  → else [current session model]
  → try complete() each until success
  → all fail → toast / overlay error: reconfigure models in F2
```

### 调用链（btw）

```text
/btw q → SendBtw → Effect::SendBtw(+models[])
  → ACP x.ai/btw { sessionId, question, models[] }
  → adapter handle_btw_request
  → /__pi_grok_btw JSON args
  → extension: snapshot branch → strip incomplete tools → system-reminder side Q
  → complete() 链式 models
  → custom pi-grok-btw/v1 { ok, answer|error, modelUsed? }
  → adapter oneshot 回 ACP result.answer
  → handle_btw_response → Done/Error overlay
```

### recap 改造

- `dispatch_send_recap` / `Effect::SendRecap` / `x.ai/recap` 传 `models: string[]`（兼容旧 `model` 单值）
- extension 对 `models` 链式 `complete()`；全失败时 manual 路径 toast「reconfigure F2 recap models」

### 冲突包

```toml
[features.pi_btw]
reason = "same role as native /btw"
packages = [
  "npm:pi-btw",
  "npm:@narumitw/pi-btw",
  "npm:@juicesharp/rpiv-btw",
]
```

## 非目标

- 不持久化 btw 到 Pi 会话 LLM 历史
- 不实现 Grok shell 的 `btw_history.jsonl`（可后补）
- 不改 Pi RPC 协议
