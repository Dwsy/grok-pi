# Issue: Pi 主题加载适配

## Goal

让 `grok-pi` 原生 Grok Pager 能发现、加载、预览并持久化 Pi 主题 JSON（与 `pi-main` 相同格式与主要发现路径），映射为 Grok `Theme` 语义色板，视觉美观可用。

## Architecture

- **Loader / map / registry**: `xai-grok-pager-render::theme::pi`
- **Runtime custom palette**: `theme::cache` + `Theme::apply_custom`
- **UX**: Grok `/theme` slash + `[ui].theme = "pi:<name>"`
- **Not in** `pi-grok-adapter`（保持 headless）

## Phases

| Phase | Scope | Status |
|---|---|---|
| A | JSON parse, vars, map, embedded dark/light, unit tests | done |
| B | Custom palette slot in theme cache | done |
| C | Discovery, `/theme`, startup, persist | done |
| D | FEATURE_MATRIX + verify | done |

## Verification

- `cargo test -p xai-grok-pager-render --lib theme::pi` → 12 passed
- `cargo test -p pi-grok-adapter --lib` → 29 passed
- `cargo check -p xai-grok-pager-bin --bin grok-pi`

## Acceptance

1. `pi:dark` / `pi:light` and `~/.pi/agent/themes/*.json` selectable via `/theme`
2. Built-in Grok themes + `auto` unchanged
3. `theme = "pi:…"` survives restart
4. Invalid JSON does not crash
5. No theme code in `pi-grok-adapter`
