# Issue: 迁移 Pi 会话恢复与树

## Goal

在不引入第二套 TUI 的前提下，让 Grok 生产 Pager 原生呈现并操作 Pi 的会话恢复与分支树；Pi 始终拥有会话文件、树、叶节点和切换逻辑。

## 背景/问题

当前 adapter 已通过 `get_messages` 回放当前 Pi 会话，但未映射 Pi RPC 的 `switch_session`、`get_entries`、`get_tree`。外部 Pi profile 因而无法在 Grok 原生表面恢复其他会话或浏览/切换 Pi 的分支树。

## 验收标准 (Acceptance Criteria)

- [ ] WHEN 用户从 Grok 原生恢复入口选择 Pi 会话，系统 SHALL 调用 Pi `switch_session` 并以新的 Pi session 重新加载原生 scrollback、标题、模型与命令目录。
- [ ] WHERE 用户打开会话树，系统 SHALL 只展示 Pi `get_tree` 返回的树与 leaf 状态，并通过 Pi 的既有会话操作切换分支。
- [ ] IF Pi 拒绝或取消会话切换，THEN 当前 Grok 视图与 Pi adapter 状态保持不变，并显示原生错误/取消反馈。
- [ ] Pi adapter SHALL 保持 library-only，且不得创建或渲染自定义终端界面。

## 关键决策

| 决策 | 理由 |
|------|------|
| 不改 Pi RPC/core | Pi 是 session 文件、树与 leaf 的唯一所有者；adapter 只扫描 metadata 与转发既有 RPC。 |
| `/resume` 与 Ctrl+S 使用现有 Grok SessionPicker | 保留生产 Pager 的既有搜索、筛选、键盘和渲染，而非新造选择器。 |
| `/tree` 是 external profile 的窄命令 | Grok 没有当前 Pi leaf tree 的现成入口；命令只打开复用 Pager list/modal primitive 的语义表面。 |
| tree 交互以 Pi 原版 TUI 为准 | 原版 `/tree` 调用 `session.navigateTree(entryId, options)`；该方法未暴露在当前 Pi RPC，且用户禁止修改 Pi 源码，因此不能用 `fork` 替代。 |

## Notes

- `--session-dir` 与 `PI_CODING_AGENT_SESSION_DIR` 现在由 composition binary 解析；未显式传参时还会从当前 Pi `sessionFile` 推断 settings 配置的非默认目录，catalog 不再错误扫描默认 agent 目录。
- 实时 assistant 终态 error 在已输出 delta 时会进入 Grok 原生 error toast；history replay 的 edit/write ToolCall 会携带 ACP diff。
- `/tree` 继续阻塞：现有 Pi RPC 可返回 tree，但不能复用 Pi 原版 `navigateTree` 语义安全切换 leaf，不能用 `fork` 伪造。
