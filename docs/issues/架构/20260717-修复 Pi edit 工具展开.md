# Issue: 修复 Pi edit 工具在 GrokTUI 中无法展开

## Goal

让 Pi 内置 `edit` 工具以原生 Grok Pager 的可展开 Edit/diff 卡片显示，并保留多处替换的全部差异。

## 已确认根因

Pi 当前 `edit` 参数为 `{ path, edits: [{ oldText, newText }] }`，而适配器
`edit_diff_content()` 仅识别已废弃的顶层 `{ oldText, newText }`。因此 `edit` 启动事件没有
ACP `Diff` 内容，Pager 生成的 Edit block 没有 hunk，依据 `is_foldable()` 无法展开。

`write` 使用 `{ path, content }`，正好落入现有适配器支持的分支，所以会被归类为 Edit 且可展开。

## 验收标准

- [ ] 单个 Pi `edit.edits[]` 替换生成可展开的原生 Edit/diff 卡片。
- [ ] 多个 `edit.edits[]` 替换均出现在展开内容中。
- [ ] `write` 继续以 Creating Edit/diff 卡片显示。
- [ ] 旧式顶层 `oldText` / `newText` 调用保持兼容。
- [ ] 适配器保持 headless，未引入 Pager UI 依赖。

## 实施计划

1. 在适配器测试中覆盖当前 `edit.edits[]` 契约并确认当前实现失败。
2. 将每个 replacement 转为 ACP `Diff` 内容。
3. 让 Pager 聚合多个 Diff 的 hunk，而非只消费第一项。
4. 运行 `cargo test -p pi-grok-adapter` 和 Pager 相关测试/检查，审查最小 diff。
