Remote TUI渲染流程

在 pi --mode rpc 下，ctx.ui.custom 是永远 reject 的空操作。 本项目不改 Pi 源码将其恢复——组件在 Pi 进程内渲染成文本帧，经 setWidget 向上投影到 Grok Pager；按键经临时 keyfile 旁路向下回传。

SYNOPSIS
render ─▶ 组件 → setWidget → Pager
input  ◀─ 键盘 → keyfile → 组件
file   ·· $TMPDIR/*.jsonl（不走 RPC）

03

关键设计点

每个环节都在守住架构不变量：不改 Pi 源码、不新建第二个 TUI。

设计01
组件运行位置
在 Pi 子进程进程内运行，复用 Pi 真实组件逻辑，行为与交互式 Pi 完全一致。
设计02
帧传输通道
复用已有 setWidget 通道，不新建 TUI，由原生 Pager 作为唯一渲染者。
设计03
多行帧保留
Pager 对 remote_tui key 特判，富组件多行帧不会被压成状态条一行。
设计04
输入旁路
无法给 Pi 加自定义 stdin RPC，改用临时 keyfile（JSONL）文件系统旁路回传按键。
设计05
光标标记剔除
剔除 CURSOR_MARKER——Pager 自己渲染帧，转发会泄漏 pi:c payload。
设计06
键盘武装
remote_tui_id 存在即拦截按键，使其流向组件而非 prompt 编辑器；Esc 优先走 cancel。
默认开启 · PI_GROK_REMOTE_TUI=1 · 设为 0 关闭 渲染走官方 setWidget 向上投影 · 输入走 keyfile 旁路向下回传
